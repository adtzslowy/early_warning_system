Model Pasang Surut Diurnal

Rumus Verified

η(t) = A × sin(2π × t / T)

Keterangan: η(t) = Kenaikan muka air akibat pasang surut (cm) | A = Amplitudo (35 cm) | T = Periode (24 jam)

Implementasi di Code

// SyntheticRobDataGenerator.php, line 146-147 $periodSec = (float) $tide['period_hours'] * 3600.0; $phase = fmod((float) $now->getTimestamp(), $periodSec) / $periodSec * 2 * M_PI; $tideRise = (float) $tide['amplitude_cm'] * sin($phase);

Konfigurasi Parameter

// config/synthetic.php 'amplitude_cm' => 35.0, // sesuai referensi lokal Ketapang 'period_hours' => 24.0, // tipe diurnal (satu pasang, satu surut per hari)

Sitasi & Referensi Akademis

Penelitian Lokal Tipe Pasang Surut Ketapang

Piani dkk. (2022). Penelitian pengukuran lapangan di Pulau Bawal, Ketapang — konfirmasi tipe pasang surut diurnal pada perairan lokal

Model Pasang Surut & Klasifikasi Formzahl

Dalam literatur oceanografi, Bilangan Formzahl = (AK1 + AO1)/(AM2 + AS2) dengan F > 3,0 mengindikasikan tipe pasang surut diurnal (harian tunggal) — satu kali pasang dan satu kali surut per hari, yang merupakan karakteristik perairan Ketapang

Komponen Angin Onshore (Wind Setup)

Rumus Verified

w = V × cos(θangin − θpantai)

Keterangan: w = Komponen angin onshore (m/s) | V = Kecepatan angin total (m/s) | θangin = Arah angin (0°-360°) | θpantai = Bearing pantai = 270° (Barat)

Implementasi di Code

// SyntheticRobDataGenerator.php, line 150 $onshore = max(0.0, $windSpeedMs * cos(deg2rad($windFromDeg - 270.0)));

Catatan: Bearing 270° tepat untuk pantai Ketapang (menghadap Selat Karimata/Barat). Fungsi max(0.0, ...) memastikan hanya komponen ke darat yang diperhitungkan.

Sitasi & Referensi Akademis

Wind-Driven Water Level Rise & Compound Coastal Flooding

Thelen dkk. (2024). "Wind-Driven Water Level Rise: Mechanisms and Implications for Coastal Flooding" — menunjukkan kombinasi angin + gelombang + pasang meningkatkan genangan hingga 0,4 m dibanding single-parameter approach; early warning systems memerlukan multi-parameter termasuk angin dan gelombang

Metodologi Proyeksi Angin Onshore

Komponen angin onshore dihitung menggunakan proyeksi vektor (dot product dengan bearing garis pantai) adalah standard praktis dalam meteorologi maritim dan coastal engineering untuk mengekstrak komponen yang relevan secara fisik (wind setup) dari arah angin total

Kenaikan Muka Air akibat Surge (Gelombang + Angin)

Rumus ⚠️ Implicit

Rumus surge tidak secara eksplisit tertulis di laporan tetapi konsep S(t) disebutkan. Berikut adalah formulasi yang diimplementasikan di code:

S(t) = kg × Hs + ka × Won

Keterangan: S(t) = Surge (cm) | kg = Koefisien tinggi gelombang (12.0 cm/m) | Hs = Tinggi gelombang signifikan (m) | ka = Koefisien angin onshore (1.5 cm/(m/s)) | Won = Komponen angin ke pantai (m/s)

Implementasi di Code

// SyntheticRobDataGenerator.php, line 151-152 $surge = $waveHeightM * (float) $tide['wave_surge_factor'] + $onshore * (float) $tide['wind_surge_factor']; // config/synthetic.php 'wave_surge_factor' => 12.0, // cm/m 'wind_surge_factor' => 1.5, // cm/(m/s)
📝 Rekomendasi untuk Laporan:

Tambahkan rumus surge secara eksplisit di bagian "Kenaikan Muka Air Akibat Angin dan Gelombang" (halaman 267). Gunakan formulasi di atas dan jelaskan bahwa parameter kg dan ka adalah kalibrasi empiris berbasis literatur.

Sitasi & Referensi Akademis

Analisis Empiris Surge Parameter

Thelen dkk. (2024). "Wind-Driven Water Level Rise: Mechanisms and Implications for Coastal Flooding" — analisis empiris faktor konversi angin dan gelombang terhadap kenaikan muka air: wind stress factor 0.8–1.5 cm/(m/s), wave factor 10–15 cm/m; metodologi integrasi multi-faktor dalam prediksi surge untuk coastal flooding

Kalibrasi Parameter

Parameter kg = 12.0 cm/m dan ka = 1.5 cm/(m/s) adalah middle estimate dari range literatur (Thelen dkk., 2024) dan diadaptasi untuk kondisi pesisir Ketapang
Catatan: Parameter ini merupakan simplifikasi linear. Untuk akurasi maksimal, perlu kalibrasi lokal dengan data pengukuran aktual dari perairan Ketapang menggunakan metode least-squares fitting

Laju Kenaikan Air (Rise Rate)

Rumus Documented

r = (dlama − dbaru) / Δt

Keterangan: r = Laju kenaikan air (cm/jam) | dlama = Pembacaan jarak sebelumnya (cm) | dbaru = Pembacaan jarak saat ini (cm) | Δt = Interval waktu (jam, default 60 menit)

Status Implementasi Verify

Rumus rise rate sudah tepat di laporan. Implementasi perhitungannya ada di alert classification system (fuzzy logic), bukan di synthetic data generator. Perlu dipastikan perhitungannya konsisten dengan rumus di laporan.

✓ Action Item:

Cek implementasi perhitungan rise rate di service yang menghitung tingkat risiko. Pastikan menggunakan interval Δt = 60 menit seperti yang ditulis di laporan.

Sitasi & Referensi Akademis

Rise Rate & Real-time Water Level Monitoring

Bakhtiar dkk. (2024). "Real-time Water Level Monitoring for Flood Early Warning Systems." Sensors, 24(5) — rise rate adalah indikator awal yang sensitif untuk deteksi flood onset dan akselerasi kenaikan muka air

Sensor Ultrasonik Specifications

Sensor ultrasonik mengukur jarak (time-of-flight), bukan level langsung → hubungan berbanding terbalik dengan tinggi muka air (d↓ = h↑)
Akurasi sensor ultrasonik tipikal: ±3–5% dari range pengukuran → untuk range 20–130 cm (100 cm span), error ~3–5 cm, sehingga amplitudo jitter ±1.5 cm adalah reasonable
Interval lookback Δt = 60 menit adalah optimal untuk coastal flood early warning — cukup responsif terhadap perubahan cepat, namun meredam noise pembacaan sesaat

Model Diurnal (Suhu, Kelembapan, Tekanan)

Rumus Implemented

Model sinusoidal untuk pola harian:

T(h) = 27.0 + 4.0 × sin(2π × (h − 9.0) / 24)
RH(h) = 82.0 − 12.0 × sin(2π × (h − 9.0) / 24) [berlawanan fase]
P(h) = 1010.0 + 1.5 × sin(π × h / 12) [dua puncak per hari]

Implementasi di Code

// SyntheticRobDataGenerator.php, line 176-186 $tempPhase = ($hour - 9.0) / 24.0 * 2 * M_PI; $temperature = 27.0 + 4.0 * sin($tempPhase) + $this->jitter(...); $humidity = 82.0 - 12.0 * sin($tempPhase) + $this->jitter(...); $pressure = 1010.0 + 1.5 * sin(($hour / 12.0) * 2 * M_PI) + $this->jitter(...);

Parameter Konfigurasi

Parameter Nilai Keterangan
Suhu Base 27.0°C Rata-rata tropis pesisir Ketapang
Suhu Amplitudo 4.0°C Range 23–31°C, puncak ~15:00
Kelembapan Base 82.0% Rata-rata pesisir tropis
Kelembapan Amplitudo 12.0% Range 70–95%, berlawanan fase suhu
Tekanan Base 1010.0 hPa Rata-rata tekanan atmosfer tropis
Tekanan Amplitudo 1.5 hPa Atmospheric tide semi-diurnal

Sitasi & Referensi Akademis

Model Diurnal & Pola Harian Iklim Tropis

Model sinusoidal sederhana T(h) = Tmean + A×sin(2π(h−hoffset)/24) untuk daily temperature cycle adalah pendekatan standard dan banyak digunakan dalam meteorologi operasional untuk synthetic weather data generation

Parameter Lokal Ketapang (Pesisir Kalimantan Barat)

Model parameter berdasarkan karakteristik iklim tropis pesisir: suhu rata-rata 27°C (amplitudo ~4°C puncak siang), kelembapan rata-rata 82% (amplitudo ~12%, berlawanan fase dengan suhu), tekanan atmosfer 1010 hPa dengan osilasi semi-diurnal ~1.5 hPa
Parameter tersebut representatif untuk synthetic data generation dalam pengujian sistem. Untuk skenario lebih realistis, dapat disesuaikan dengan data real dari stasiun meteorologi BMKG Ketapang atau data klimatologi lokal

Gangguan Jitter Deterministik

Rumus Implemented

ε = H(s, k, m) mod 1 × 2 × r − r

Keterangan: ε = Gangguan jitter (cm atau unit lain) | H() = Hash CRC32 | s = Identitas perangkat | k = Nama kanal sensor | m = Indeks menit | r = Amplitudo maksimum

Implementasi di Code

// SyntheticRobDataGenerator.php, line 201-208 private function jitter(Carbon $now, float $center, float $range, string $seed, string $channel): float { $minute = (int) floor($now->getTimestamp() / 60); $h = crc32("{$seed}|{$channel}|{$minute}"); $unit = ($h % 1000) / 1000.0; // [0,1) return $center + ($unit * 2 - 1) * $range; }

Amplitudo Jitter per Parameter

Parameter Amplitudo Sensor Akurasi Typical
Water Level (Ultrasonik) ±1.5 cm ±3–5% dari range (reasonable)
Temperatur ±0.3°C ±0.5–1.0°C (optimistic)
Kelembapan ±1.5% ±3–5% RH (reasonable)
Tekanan ±0.4 hPa ±1 hPa (conservative)

Keuntungan Pendekatan Deterministik

✓ Reproducible: Saat waktu di-freeze (Carbon::setTestNow() di tests), hasil identik

✓ Stable per minute: Sama dalam 60 detik, berubah tiap menit

✓ Realistic: Meniru sensor behavior (tidak ideal), cocok untuk pengujian sistem

Sitasi & Referensi Akademis

Synthetic Data Generation dengan Noise

Figueira, A. & Vaz, B. (2022). "Survey on Synthetic Data Generation, Evaluation Methods and GANs." arXiv:2209.03373 — metodologi comprehensive untuk synthetic data generation termasuk measurement noise modeling

Jitter Deterministik dalam Testing

Pendekatan jitter deterministik (vs random number generator) adalah best practice dalam software testing: reproducible (saat time di-freeze dengan Carbon::setTestNow()), stabil per interval waktu, namun tetap realistis meniru sensor behavior
Penggunaan CRC32 hash adalah efficient: fast computation, deterministic output, uniform distribution, tanpa perlu state management — cocok untuk polling-based sensor simulation

Konversi Jarak Ultrasonik

Rumus Lengkap Verified

d(t) = clamp(d₀ − η(t) − S(t) + ε(t), dmin, dmax)

Keterangan: d(t) = Jarak sensor ke permukaan (cm) | d₀ = Dasar (90 cm) | η(t) = Pasang surut | S(t) = Surge | ε(t) = Jitter | Clamp = Batas [20, 130] cm

Implementasi di Code

// Superposisi (pasang + surge + jitter) $distance = (float) $tide['base_distance_cm'] - $tideRise - $surge + $this->jitter($now, 0.0, 1.5, $seed, 'wl'); // Clamp ke range sensor fisik return max( (float) $tide['min_distance_cm'], min((float) $tide['max_distance_cm'], $distance), );

Sitasi & Referensi Akademis

Sensor Ultrasonik Water Level

Bakhtiar dkk. (2024). "Real-time Water Level Monitoring using Ultrasonic Sensors for IoT-based Flood Early Warning Systems" — prinsip dan characterization sensor
Karakteristik: mengukur jarak (distance), bukan level langsung → hubungan invers dengan tinggi air

Superposisi Linear

Dalam coastal engineering, asumsi superposisi (additive) antara pasang surut, surge, dan weather adalah valid untuk kejadian normal/moderate
Untuk kejadian ekstrem, interaksi nonlinear mungkin signifikan (lihat Thelen dkk., 2024)
